# LFM Trope

The `lfm` Trope is the workflow that produces and maintains
Literate Framework Manifests. This file is the imperative
counterpart to `concept.mdx`'s typed contract: what a Step
sequence does to author one LFM and to maintain references when
hashes change.

The procedure decomposes into four atomic Steps. The composition
is a `workflowStep` whose `run` is `Effect.gen` sequencing the
atomic Steps under `memo()`. The Trope is one entry in the
broader Trope category; an LFM file is one invocation of this
Trope.

## Author body

Take an `LFMAuthoring` input and emit a body string. The body
must be standalone declarative prose: present-tense statements
of current-state, no supersession references, no narrative
cross-LFM dependence. Soft `@lfm(<short-hash>)` annotations are
permitted as operational tooling — they must not be the
grammatical subject of any clause.

Step: `effect`. Input: `LFMAuthoring`. Output: `RawBody`
(string). No services — pure construction.

The Trope's prose authority for *what* to write lives in this
file's parent — the `lfm` Concept's `concept.mdx`. This Step is
the mechanical assembly of validated input into a body string.
For agent-driven authoring, the Step's input itself is what an
agent produces under the gate; the Step is then an identity
transformation.

## Compute id

Compute the short content hash of the body. SHA-256 of the body
bytes (UTF-8); take the first 8 hex characters.

Step: `effect`. Input: `RawBody`. Output:
`{ body: string, id: string }`. No services — pure.

The hash *excludes* the metadata header so that operational
fields (`status`, written by `reconcile`) do not trigger
identity changes. This Step is invoked on a header-less body.

## Validate self-sufficiency

Run the two heuristic checks from `concept.mdx#self-sufficiency-validation`:

1. Detect prose patterns that make a soft link a grammatical
   subject (`as declared in @lfm(...)`, `per @lfm(...)`).
2. Detect markdown links to LFM file paths
   (`[..](./other-domain.md)` patterns under `corpus/manifests/`).

Each match contributes a `Violation` to the report. The Step
returns the report; v0.1 emits violations as warnings (not
errors) so the Trope can be invoked on partial migrations.

Step: `effect`. Input: `{ body: string, id: string }`. Output:
`{ body, id, violations: readonly Violation[] }`. No services —
pure.

## Write file

Serialise the metadata header (YAML between `---` fences) above
the body. Write to `corpus/manifests/<layer-path>/<domain>.md`.

When this is an *update* (the LFM existed previously with a
different id), record the old id so the caller can invoke
`updateReferences(oldId, newId)`. The Trope itself does not
walk the corpus to update references — that is the
`reconcile` Trope's contract. This Step writes the single LFM
file.

Step: `io`. Input:
`{ body, id, violations, layer, domain, disposition, dependencies?, oldId? }`.
Output: `LFMRef` (path + new id + oldId | null). Services:
`SessionStore`.

## Update references

A separately-callable Step (not in the main composition; called
by `reconcile`). Walk every LFM under `corpus/manifests/`,
rewrite `@lfm(<oldId>)` annotations to `@lfm(<newId>)` (or
remove them when `newId` is `null`, signalling deletion),
write back any LFM whose body changed.

Files inside `corpus/sessions/*` are excluded — they are
immutable historical record.

Step: `io`. Input: `{ oldId: string, newId: string | null }`.
Output: `{ updated: readonly string[] }` (paths whose bodies
were rewritten). Services: `SessionStore`.

## Composition

The Trope's `realise` is a `workflowStep` whose `run` is:

```typescript
Effect.gen(function* () {
  const body = yield* memo(authorBody)(input)
  const hashed = yield* memo(computeId)(body)
  const validated = yield* memo(validateSelfSufficiency)(hashed)
  const ref = yield* memo(writeFile)({
    ...validated,
    layer: input.layer,
    domain: input.domain,
    disposition: input.disposition,
    dependencies: input.dependencies,
  })
  return ref
})
```

The four primary Steps each have their own `ProseRef` with
`section: 'author-body' | 'compute-id' | ...`. The fifth
(`update-references`) is exposed separately on the Trope module
for the `reconcile` Trope to import.
