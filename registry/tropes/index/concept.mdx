# Index Procedure

The `index` Trope produces a permanent corpus index — a single
markdown document at `corpus/manifests/index.md` summarising
every LFM in the corpus. The index is a **navigation surface**:
it carries no decisive content of its own, references LFMs via
soft `@lfm(<short-hash>)` annotations, and is regenerated on
every invocation.

## Disposition

`Protocol`. Mechanical; no AI in the loop.

## Optional, default-shipped

The `index` Trope is **optional**: a consumer with no need for a
single navigation file can omit it from their template. It is
**default-shipped** in the `minimal` template seed list so most
consumers get it out of the box and discover it without having
to opt in.

A consumer who decides the index is noise can delete the seed
and the file; nothing else in LF requires the index to exist.

## Input

```typescript
interface IndexInput {
  readonly repoRoot: string
}
```

## Output

A side-effect: the file `corpus/manifests/index.md` is rewritten
atomically. The Step also returns:

```typescript
interface IndexResult {
  readonly indexPath: string
  readonly entriesIncluded: number
}
```

so callers can chain.

## Index document shape

```markdown
# Manifest Index

| Layer / Domain | id | status | summary |
|---|---|---|---|
| `protocol/algebra` | `a3f2c8b1` | Reconciled | LF declares a four-level algebra: Concept → Trope → Step → instance. |
| `protocol/session-lifecycle` | `def01234` | Reconciled | Sessions are Open / Closed / Planned / Abandoned … |
| `workspace/namespace` | `5eaf6789` | Drifted | LF rewrite packages live under `@literate/*` … |
| ...
```

The summary cell is the LFM body's first sentence (everything
between the first `# heading` and the first `\n\n` after it,
trimmed). The Trope does not invent prose; it lifts existing
prose from the LFM body.

The index does **not** carry decisive content of its own. If a
reader wants to know what the framework declares about a
Domain, they read the LFM. The index points; it does not
declare.

## Walk order

Same as `reconcile`: `workspace/`, `apps/<app>/`,
`infrastructure/[<target>/]`, `protocol/`. Within each Layer,
domains alphabetical.

## Idempotence

Two consecutive `index` invocations on the same world produce
identical output. The Trope is safe to chain after `reconcile`
(typical pattern: a CLI verb might run `reconcile` then
`index`).

## Errors

- `IndexWriteFailed { path, reason }` — the file write failed
  (disk full, permissions). The error is surfaced; no other
  state changes.
- `ManifestsAbsent { repoRoot }` — `corpus/manifests/` does
  not exist. The Trope returns a no-op result; no index file
  is written.

## Services required

- `SessionStore` — directory listing, file read/write.

## Relationships

- **Realises** the `index` Concept.
- **Reads** every LFM under `corpus/manifests/`. **Does not
  call** the `lfm` Trope or the `reconcile` Trope; the index is
  pure navigation.
- **Optionally invoked by** an aggregating CLI verb that
  chains `reconcile` and `index`.

## Versioning

- `version: '0.1.0'` at introduction.
- Adding columns to the index (e.g., last-modified) is a
  minor bump. Removing columns or changing the file location
  is major.
