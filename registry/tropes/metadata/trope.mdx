# Metadata Trope

The `metadata` Trope owns the parse / serialise round-trip for
typed head-of-file metadata blocks. It binds to the `metadata`
Concept (the typed substrate-level Record<string, string>); the
contract lives in that Concept's `concept.mdx`. This file is the
imperative decomposition the Step composition implements.

The Trope's principal surface is its **pure helpers**, not its
realise. Reconcile imports `parseMetadataBlock`,
`serialiseMetadataBlock`, `parseMetadataDirective`, and
`parseYamlFrontmatter` directly to do its YAML → directive
migration and ongoing read/write of metadata blocks. The realise
itself is a minimal validation pass — Metadata is largely
declarative, and the heavy lifting is in the parse / serialise
pair.

## Parse

`parseMetadataBlock(content)` — accept either wire form at the
head of the input. Try the canonical directive form
(`::metadata{...}`) first; fall back to the legacy YAML
(`---`-fenced) form. Return `null` when neither matches.

The directive parser walks attribute payload depth: top-level
commas separate `key=val` pairs, but commas inside braces,
brackets, or quoted strings stay verbatim. Single-quoted values
strip a single layer of quotes; JSON-ish object literals
(`{base:'Protocol', scope:'algebra'}`) round-trip unchanged.

The YAML parser is the legacy line-by-line key/value extractor
preserved for backward compatibility during the corpus
migration.

Both forms return the same `ParsedMetadataBlock` shape:

```typescript
interface ParsedMetadataBlock {
  readonly form: 'directive' | 'yaml'
  readonly meta: Record<string, string>
  readonly headerRaw: string
  readonly body: string
}
```

The `form` field lets callers know which wire form they
encountered (useful for migration: a yaml-form parse can be
rewritten as a directive-form serialise).

## Serialise

`serialiseMetadataBlock(meta)` — emit the canonical
`::metadata{...}` directive form. Keys are ordered: the
canonical six (`id`, `disposition`, `layer`, `domain`, `status`,
`dependencies`) come first in declaration order; extra keys
follow in insertion order.

Values that are bare tokens round-trip without quotes; values
containing whitespace, commas, or quote characters get
single-quote wrapping. JSON-ish object / array literals stay
verbatim — the directive splitter walks brace / bracket depth so
they parse back correctly.

Empty-string values are omitted entirely. Callers who want to
record the absence of a value should omit the key from the
record rather than passing an empty string.

## Validate

The `validateMetadataStep` is the round-trip identity: take a
`{ meta }` input typed by `MetadataSchema`, return the same
`{ meta }`. The Schema validation is the contract — successful
validation guarantees the record is a `Record<string, string>`.

This Step is intentionally minimal. Metadata is declarative;
there is no semantic enforcement at the substrate level (LFM
fields, status transitions, etc. are downstream consumer
contracts).

## Composition

The Trope's `realise` is a `workflowStep` whose `run` is:

```typescript
Effect.gen(function* () {
  return yield* memo(validateMetadata)(input)
})
```

One atomic Step. The composition is trivial because the
substrate-level work is parse / serialise, exposed as pure
helpers callers (notably reconcile) import directly.
