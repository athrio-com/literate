# Goal Status

The closed vocabulary of `Status:` values a session-Goal can
carry. A Goal is a session-scope specification that names the
work and its upstream prose. `Status:` is added on Accept of
the Goal at the gate (not present in the draft presented for
gating).

## Members

- **`Active`** — being worked on now. The post-Accept default.
- **`Completed`** — Acceptance criteria met. Set atomically
  on completion; ungated.
- **`Abandoned`** — retired without completion. Gated
  (authorial retirement, not mechanical).
- **`Superseded by Goal N`** — replaced by a later Goal in
  the same session. Set atomically when the new Goal is
  Accepted; ungated. Preserves the journal of intent (IMP-3).

## Schema shape

```typescript
GoalStatus =
  | 'Active' | 'Completed' | 'Abandoned'
  | `Superseded by Goal ${number}` (open string family)
```

The Effect Schema definition is a `Schema.Union` of the three
closed literals and a `Schema.String` constrained by
`Schema.pattern(/^Superseded by Goal \d+/)`.

## Transitions

```
Active → Completed
Active → Superseded by Goal N
Active → Abandoned         (gated)
```

Terminal states do not transition further.

The `session-end` Trope's `validateStep` reads the `Status:`
line of every Goal and refuses close when any value is
non-terminal (i.e. is `Active`). The terminal-status check is
exposed as `isTerminalGoalStatus(raw)` from this Concept's
`index.ts` — `session-end`'s validator imports the predicate
rather than re-implementing the closed set.

## References

- Used in every session log's `## Goals` section in
  `corpus/sessions/`.
- The `session-end` Trope's `validateStep` calls
  `isTerminalGoalStatus` for each Goal; failures surface as
  `Goal[<n>].terminal-status` entries in `SessionEndIncomplete.missing`.
- IMP-3 (during-session scope-drift gate) writes
  `Superseded by Goal N` atomically when a new Goal supersedes
  a prior one.
