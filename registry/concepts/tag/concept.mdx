# Tag

A **Tag** is a slugged label used on ADRs (and forward, on
other authored prose — chapters, specs, memos) for
cross-cut search and conflict detection. Tags are flat: there
is no hierarchy or implication. Two ADRs share relevance iff
they share a tag.

## Type vs instance

The Tag *type* (this Concept) is a **Protocol mechanism** —
every LF user has tags. The specific tag *set* a project uses
(its authored slug enumeration) is **consumer content**, not
part of this Concept's Schema. LF's own slug set lives in
`corpus/tags.md`; consumers author their own at the same
relative path in their corpus. This split mirrors the deeper
principle from ADR-031 / ADR-033 (typed Concepts ship; specific
authored instances stay in their author's corpus).

## Shape

A tag is a `#` followed by a lowercase ASCII slug:

```typescript
type Tag = `#${string}`  // narrowed by Schema.pattern
```

The Effect Schema definition is:

```typescript
TagSchema = Schema.String.pipe(
  Schema.pattern(/^#[a-z][a-z0-9-]*$/, {
    message: () => 'tag must match `#[a-z][a-z0-9-]*`',
  }),
  Schema.brand('Tag'),
)
```

Brand-typing prevents accidental confusion with arbitrary
strings; `Tag` is a nominal subtype of `string`.

## Closed-set membership per consumer

The Schema enforces only the slug shape, not the slug set. A
consumer that wants typed enforcement of their authored slug
set composes the helper exposed by this Concept's `index.ts`:

```typescript
import { closedTagSet } from '.../tag'

const ProjectTagSet = closedTagSet([
  '#process', '#algebra', '#protocol',
  // ...consumer's authored set...
] as const)
// ProjectTagSet: Schema.Schema<'#process' | '#algebra' | '#protocol'>
```

Alternatively, a forward `tag-flow` Trope can read the slug
set from `corpus/tags.md` at gate time and pass it through the
gate — defer to consumer preference.

## Authoring the slug set

Each consumer authors their own slug set at `corpus/tags.md`
(or wherever their convention places it). Adding a tag is a
gated authorial change — it requires a Goal-or-equivalent
gate because tags carry meaning that propagates through every
ADR carrying them. Removing a tag requires a separate gated
change and retagging affected ADRs.

LF's own slug set as of this Concept's authoring includes
`#process`, `#algebra`, `#protocol`, `#corpus`, `#tooling`,
`#licensing`, `#self-hosting`, `#release`, `#site`,
`#template`, `#execution`, `#migration`. See LF's
`corpus/tags.md` for the canonical set + per-tag semantics.

## Morphisms

Tags are flat; there is no hierarchy or implication. Two ADRs
share relevance iff they share a tag. An ADR carrying only a
single tag signals a narrow concern.
