# ADR-030 — Adopt `@effect/cli` for the CLI's argv surface

**Date:** 2026-04-24
**Status:** Accepted
**Tags:** `#tooling` `#release`

**Context:**

ADR-004 (2026-04-22) committed the CLI to `@effect/cli` on top of
`effect` and `@effect/platform`. The post-ADR-025 P2 push shipped
the CLI with a hand-rolled `switch(verb)` dispatcher and ad-hoc
`--flag` parsing inside each verb module — a deliberate
shortcut to land the shadcn-shaped distribution before adopting
the full Effect-CLI machinery. ADR-028 (2026-04-24) closed the
follow-up Effect-depth gap by making every verb's `run` return
`Effect<number, unknown, VerbRequirements>` and promoting
registry + weaver services to `Context.Tag`s with Layer
factories. With verbs Effect-composed end-to-end, `Command.make`
+ `Options` + `Args` are a structurally trivial replacement for
the hand-rolled argv code; the only reason this hadn't already
happened was sequencing.

This ADR records the completion: the argv surface now uses
`@effect/cli` as ADR-004 originally committed.

**Decision:**

The CLI's argv surface is built with `@effect/cli`. Concretely:

- `packages/cli/src/bin/literate.ts` is a single `Command.run`
  invocation against a root `Command.make('literate', …)` that
  composes the six verbs as subcommands via `Command.withSubcommands`.
  No hand-rolled `switch(verb)`. No hand-rolled `--flag` loops.
  No bespoke `usageBanner()` / per-verb `usage` strings.
- Each verb module under `packages/cli/src/verbs/` exports a
  `Command.make`-built command (default export) alongside its
  programmatic `runX` helper. Options use `Options.text` /
  `Options.boolean` / `Options.optional` / `Options.withDefault` /
  `Options.withAlias`. Args use `Args.text` / `Args.choice` /
  `Args.optional`. Singular-form aliases (`trope` → `tropes`,
  `concept` → `concepts`) live in the `Args.choice` value list
  so the normalisation runs in `@effect/cli`'s parser, not in
  hand-rolled verb code.
- `--help` and `<verb> --help` output is generated by
  `@effect/cli`. Per-verb descriptions feed `Command.withDescription`.
- The runtime is provided by `@effect/platform-bun`'s
  `BunContext.layer` (FileSystem / Path / Terminal) and the
  process exit / signal handling by `BunRuntime.runMain`. The
  ADR-029 Bun-only runtime narrowing makes `@effect/platform-bun`
  the single platform-implementation dep needed.
- `@effect/cli`, `@effect/platform`, and `@effect/platform-bun`
  land as runtime `dependencies` of `@literate/cli`, not
  devDependencies — they execute inside the bundle.
- The previously-needed argv-boundary tagged errors
  (`UsageError`, `UnknownVerb`, `UnknownKind`) are dropped from
  `src/errors.ts`. `@effect/cli`'s `ValidationError` shape
  replaces them; `Args.choice` rejects unknown kinds before any
  verb body runs. `UnknownTemplate` is kept because the template
  name is a free-form string not modelled as `Args.choice`.

**Boundary:**

- This ADR scopes the **argv surface**. Verb semantics, Layer
  composition, error types, and the Bun-only runtime were
  settled by ADR-028 / ADR-029 and are unchanged.
- The programmatic `runX` helpers (`runContinue`, `runClose`,
  `runTangle`, `runUpdate`, `runInit`, `runWeave`) keep their
  existing Effect-returning signatures so the e2e and
  prose-schema test suites continue to drive verbs without going
  through argv parsing.
- Two interactive surfaces `@effect/cli` enables are deferred:
  - `Prompt`-based interactive verb prompts (defer until a verb
    actually needs one).
  - Shell completion script generation (`@effect/cli` generates
    them; defer until a consumer asks).
  - Either becomes a separate session if/when needed.
- The bundle grew from 1692.6 kB (post-ADR-028) to 2783.8 kB
  with `@effect/cli` + `@effect/platform-bun` inlined. This is
  expected; the parent session noted bundle-size order of attack
  if it ever matters (remark first ≈ 800 kB, then runtime
  shims). Not a concern at v0.1.

**Rationale:**

- Restoring ADR-004's named commitment removes a chronic doc-vs-
  code drift point: ADR-004 named `@effect/cli`; the code wasn't
  using it; every reader of either had to reconcile.
- Typed `Options` / `Args` move every argv-shape constraint into
  the type system. The `kind` argument is a literal union of
  four strings; the parser enforces that before any verb body
  runs.
- Generated `--help` removes the per-verb `usage` string as a
  drift surface — the help is computed from the same declaration
  the parser uses.
- `BunRuntime.runMain` handles signal / exit-code mapping
  uniformly; the bin file no longer has bespoke
  `formatError` / `exitCodeFor` helpers.

**Consequences:**

- `packages/cli/src/verbs/registry.ts` and
  `packages/cli/src/verbs/verb.ts` are deleted — the `Verb`
  interface and `VERBS` registry are subsumed by `@effect/cli`'s
  `Command` composition. The bin's subcommand list (six
  imports + a `withSubcommands` array) is now the sole source
  of truth for which verbs the CLI exposes.
- `packages/cli/src/index.ts` no longer re-exports `VERBS` /
  `usageBanner` / `Verb`; consumers driving verbs from code
  compose against the `runX` helpers + service Layers.
- Adding a verb is now: one new file under `src/verbs/` exporting
  a `Command.make` value (default export) plus its `runX`
  helper, plus one import + one entry in
  `bin/literate.ts`'s `withSubcommands` array.

**Amends:** ADR-004 (the `@effect/cli`-on-`@effect/platform`
clause is now load-bearing in code, not aspirational; ADR-004's
`Status:` line annotated to record the completion; ADR-004 body
untouched per IMP-6).
