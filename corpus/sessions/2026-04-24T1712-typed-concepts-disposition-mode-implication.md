### Goal 1 — Disposition Concept (retires ADR-021's Modality)

**Status:** Completed
**Category:** feature
**Topic:** Author `@literate/concept-disposition` as a
parametrised struct with Effect Schema. `base` is a closed set
of three literals (`'Product' | 'Protocol' | 'Infrastructure'`);
`scope`, `prompt`, `prose` are optional open strings.
Disposition names the *referential domain* a Trope or authored
instance belongs to — replaces the term "Modality" from ADR-021
because "modality" carries the wrong linguistic weight
(epistemic / deontic / alethic, not referential-frame).

**Upstream:** ADR-021 (Modality — superseded into Disposition
here); the abandoned Planned log
`2026-04-26T0900-disposition-concept.md` for the Goal text;
exploration §8 + §13.5.

**Acceptance:**
- `registry/concepts/disposition/{concept.mdx, index.ts, README.md}`
  authored with typed Schema:
  `Schema.Struct({ base: Schema.Literal('Product', 'Protocol', 'Infrastructure'), scope: Schema.optional(Schema.String), prompt: Schema.optional(Schema.String), prose: Schema.optional(Schema.String) })`.
- `concept-session`'s schema includes `disposition` (optional).
- ADR-031 exists Accepted; ADR-021 annotated Superseded.

### Goal 2 — Mode Concept + IMP-N (Mode-discipline imperative)

**Status:** Completed
**Category:** feature
**Topic:** Author `@literate/concept-mode` with typed Schema
over the Mode ADT (`Exploring | Weaving | Tangling`). Mode
names the *operational stance* of a session, orthogonal to
Disposition. Name the enactor axis explicitly — a Mode is
enacted either by an agent-in-session (gated, AI-assisted) or
by the mechanical CLI (deterministic, non-AI). Same Mode
Concept; different enactor; different per-enactor discipline.

IMP-N is the load-bearing part — a new Mandatory Agent
Instruction binding agent behaviour to the active Mode.
Without it, agentic IDEs default to action and collapse
Exploring into premature implementation.

**Upstream:** Abandoned Planned log
`2026-04-26T1400-mode-concept-and-imperative.md`; ADR-025 §5
(enactor axis flagged); exploration §9 + §11.

**Acceptance:**
- `registry/concepts/mode/{concept.mdx, index.ts, README.md}`
  authored.
- `concept-session` carries `mode` (optional for v0.1
  backward compat).
- IMP-N landed in `corpus/CLAUDE.md` after IMP-6.
- IMP-N landed in `packages/template-minimal/files/CLAUDE.md`
  so the next `literate weave` carries it to consumers.

## Deferred / Discovered

### Deferred

- **Metalanguage migration: `Modality` → `Disposition` +
  `Mode`.** Replace `modality: Modality` (required) on
  `Trope<C>` with `disposition: Disposition` (required) +
  `mode?: Mode` (optional); replace the optional
  `modality?: Modality` on `Concept<D>` with
  `disposition?: Disposition` + `mode?: Mode`; remove the
  `Modality` ADT and its constructors from
  `@literate/core/kinds.ts`; rewrite the two existing Trope
  seeds (`session-start`, `session-end`) and the two
  existing Concept seeds (`session-start-procedure`,
  `session-end-procedure`) to drop `modality:
  Modality.Protocol` in favour of `disposition:
  Disposition.Protocol`. Risk surface: every `import {
  Modality } from '@literate/core'` site has to flip; tests
  + smoke install have to stay green throughout. Sized as
  one focused refactor session.
- **Mode-setting Step on `session-start`.** A new
  `effectStep` (or `gateStep` when ambiguous) inserted into
  the `session-start` Trope's workflow, defaulting Mode to
  the heuristic in `concepts/mode/concept.mdx` and gating
  when ambiguous. Requires synchronised edits to
  `index.ts` + `prose.mdx` + the `proseSchema.h2Slugs`
  list, plus a test for the new Step's gate path. Sized as
  one session.
- **Disposition-setting Step on `session-start`.** Parallel
  to the Mode-setting Step; defaults Disposition to
  `{ base: 'Protocol' }` for sessions authoring LF itself
  and gates the base selection otherwise. Same shape /
  same risk surface; can land in the same session as the
  Mode-setting Step.
- **Mode-transition validator on `session-end`.** Validates
  that any in-session Mode shifts were gated and
  timestamped. Requires the session log to journal Mode
  shifts in a structured form (e.g. a `## Mode History`
  section); spec the journal format in the same session as
  the session-start Mode-setting Step.
- **`trope-implication-flow` seed.** A typed Trope at
  `registry/tropes/implication-flow/{index.ts, prose.mdx,
  README.md}` providing Surface → Promote / File / Dismiss
  Steps with their typed I/O. v0.1 ships the Concept + the
  session-end validator only; the realising Trope is the
  next piece.
- **`ImplicationNotTerminal` dedicated TaggedError.** v0.1
  piggybacks on the existing `SessionEndIncomplete`
  aggregate (with `Implication[<id>].terminal-status`
  entries in `missing`). A dedicated
  `ImplicationNotTerminal { id, currentStatus }` typed
  error in `registry/tropes/session-end/index.ts` is
  cleaner long-term but not load-bearing for v0.1.
- **`Disposition.scope` promotion flow.** When a freeform
  `scope` value accretes enough weight to warrant naming,
  the Person promotes it via an ADR. The flow needs a
  Trope or a documented procedure; v0.1 leaves it
  unspecified.
- **P8** (`2026-04-27T1400-template-finalisation.md`) stays
  `Status: Planned` — the capstone session for a future
  thread (template finalisation + registry trust + MVP-2
  end-to-end smoke). This thread does not open it.

  Originally drafted to land "before the descriptive
  sections"; placed instead immediately before IMP-6 (the
  NEVER list) so the during-session imperative ordering
  (1 → N → 6) reads cleanly. Same content; the placement
  decision is journalled here.

  to mark the first authored release of the typed surface.
  Existing Concepts (e.g. `session-start-procedure` at
  `0.0.1`) follow a different convention (the rewrite-

  the live consumer-facing surface; the v0.0.x range is
  pre-release scaffolding. Worth a future session's
  prose-only commitment if the convention needs codifying.
