## Work Done

### Goal 1 — `@effect/cli` adoption

- **`packages/cli/package.json`** — added `@effect/cli` (`^0.75.1`),
  `@effect/platform` (`^0.96.1`), and `@effect/platform-bun`
  (`^0.89.0`) as runtime `dependencies`. The platform-bun dep
  provides `BunContext.layer` (FileSystem / Path / Terminal) and
  `BunRuntime.runMain`; per ADR-029 it is the single
  platform-implementation dep needed.
- **`packages/cli/src/bin/literate.ts`** — replaced the hand-rolled
  `switch(verb)` dispatcher and ad-hoc `--flag` parsing with a
  single `Command.run` invocation. The bin file is now ~50 lines:
  six subcommand imports, a root `Command.make` composed via
  `Command.withSubcommands`, `Layer.merge`-composed `CliServicesLive`
  (Config / Manifest / Fetcher / Weaver), `Command.run` building
  the CliApp, then `BunContext.layer` provided and
  `BunRuntime.runMain` mapping the program to a process exit code.
- **`packages/cli/src/verbs/tangle.ts`** — `Args.choice<SeedKind>`
  with the four-pair list inlines the singular→plural normalisation
  the previous `normaliseKind` helper did manually. `Options.text('registry')`
  with `withAlias('r')` + `optional` replaces the `--registry / -r`
  loop. Handler reads `process.cwd()`, calls `runTangle`, prints
  via `Console.log`. The exported `runTangle` Effect is unchanged.
- **`packages/cli/src/verbs/update.ts`** — same `Args.choice` shape
  as `tangle`; no flags. Handler unchanged in semantics.
- **`packages/cli/src/verbs/weave.ts`** — `Command.make('weave', {})`
  with no positional args or options; handler dispatches to the
  unchanged `runWeave` and prints the woven path + per-bucket
  inclusion lists.
- **`packages/cli/src/verbs/init.ts`** — `Args.text({ name: 'target' })`
  optional; `Options.text('template')` with `withDefault('minimal')`;
  `Options.text('registry-url')` and `Options.text('registry-ref')`
  optional with env-var fallbacks (`LITERATE_REGISTRY_URL`,
  `LITERATE_REGISTRY_REF`) resolved in the handler via
  `Option.getOrElse`. Handler dispatches to `runInit` (unchanged).
- **`packages/cli/src/verbs/continue.ts`** — `Args.text({ name: 'repoRoot' })`

- **`packages/cli/src/verbs/close.ts`** — `Args.text({ name: 'sessionPath' })`

- **`packages/cli/src/errors.ts`** — dropped `UsageError`,

### Discovered in this session

- **`@effect/cli` Args.choice with value mappings is the right
  shape for singular-form aliases.** The pre-refactor verbs had
  a `normaliseKind` helper that re-mapped `'trope'` → `'tropes'`.
  `Args.choice<SeedKind>([ ['tropes', 'tropes'], ['trope', 'tropes'], … ])`
  inlines that mapping inside the parser; the helper retired
  cleanly. Worth remembering when modelling future closed-vocab
  args.
- **`BunContext.layer` from `@effect/platform-bun` is the single
  platform-implementation dep we need.** `@effect/platform`'s
  `Terminal` + `FileSystem` + `Path` are abstract; `BunContext`
  provides Bun-native concretes for all three under one Layer.
  Per ADR-029 we don't need a Node fallback. Deferred decision
  if a future sister product ever wants the same CLI on Node:
  add `@effect/platform-node` as an alternative.
- **Help output for `Args.choice` lists every choice value in
  the inline help row.** With four kind aliases (`tropes` /
  `concepts` / `trope` / `concept`) the line gets noisy in
  `literate --help`'s top-level COMMANDS table:
  `tangle ... <kind> tropes | concepts | trope | concept <id>`.
  Acceptable for v0.1 but worth a future gloss in the per-verb
  description if it bothers anyone.
- **Bundle-size delta with `@effect/cli`** — adding the three
  `@effect/*` packages added ~1091 kB to the bundle, larger
  than the ~22 kB Bun-target delta from ADR-029 but consistent
  with the parent session's note that Effect runtime is the
  second-biggest contributor after remark. Total still under
  3 MB; not a concern at v0.1.
