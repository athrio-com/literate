# Corpus

A corpus is the prose half of every repo that uses LF. It lives at
`corpus/` in the repo root and contains the authored prose defining
what the product is and becomes: decisions, specs, chapters,
stories, sessions, categories, concepts, memos. The corpus is the
source; `src/` is the derivative; `.literate/` is the framework
snapshot that governs authoring.

The `corpus → src` derivation is one-way. Code is derived from
accepted prose. Prose never points at code; code points at prose
via traceability comments (`@adr ADR-NNN`, `@spec spec-slug`).

## Structure

At minimum a corpus contains `decisions/`, `categories/`, and
`sessions/`, each with an index file. Additional folders activate
when their corresponding Tropes are installed: `specs/`,
`chapters/`, `stories/`, `concepts/`, `memos/`.

Every folder carries an index file whose stem matches the folder
name (`decisions.md`, `sessions.md`, `categories.md`,
`concepts.md`, …). Indexes are mutable and mechanical; artefacts
obey their own mutability profile (see the framework Protocol).

## Relationships

- The `session`, `decisions`, `category`, `concept`, and
  `chapters` Concepts each instantiate inside `corpus/`.
- LF's own repo carries a corpus (`corpus/` at the root) holding
  LF-project decisions, separate from the Concepts and Tropes LF
  ships under `packages/`.

## Open questions

- Whether a corpus can span multiple root folders (e.g.,
  per-package corpora in a monorepo) is out of scope for v0.1.
