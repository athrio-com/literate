# Session Trope

The canonical realisation of the `session` Concept. Declares the
on-disk shape of session logs, the lifecycle operations
(`session-start`, `session-end`), the gating rules applied to
`## Goals` entries, and the optional `## Plan` block that pre-scopes
multi-session arcs.

## File layout

Session logs live at `corpus/sessions/YYYY-MM-DDTHHMM-<slug>.md`.
The filename encodes date and start-time to minute precision; the
slug is a short topical handle. For a `Planned` session whose start
time is not yet known, the timestamp encodes the planned start (or a
synthetic placeholder authored by the parent session); on transition
to `Open`, the file is renamed to reflect the actual start time
unless already correct. The folder index is
`corpus/sessions/sessions.md` listing logs newest first, with
Status.

## Header

```markdown
# Session: YYYY-MM-DD — topic

**Date:** YYYY-MM-DD
**Started:** YYYY-MM-DDTHH:MM | —
**Status:** Planned | Open | Closed (YYYY-MM-DDTHH:MM) | Abandoned
**Chapter:** <handle> — <title> | —
**Agent:** <model id> | —
**Planned by:** corpus/sessions/<parent-log>.md   (optional)
```

`Started` is `—` while `Status` is `Planned`. `Agent` is `—` until
a model is bound to the session at start time.

## Sections

1. `## Goals` — gated entries. Each Goal carries `Topic`,
   `Upstream`, optional `Scope`, `Out of scope`, `Acceptance`,
   `Notes`. On Accept, `Status` and `Category` are added. A
   `Planned` session may carry provisional Goals copied from a
   parent's Plan entry; they are re-gated when the session
   transitions to `Open`.
2. `## Plan` — *optional*. Present on sessions that pre-scope a
   multi-session arc. Each Plan entry carries `Slug`, `Topic`,
   optional `Depends on`, and a `Goals` list whose entries carry
   `Topic`, `Upstream`, `Acceptance`. On Accept of a Plan entry,
   a successor `Planned` session log is created and the entry is
   linked to it. When the successor transitions to `Open`, the
   entry is frozen and gains a `Realised by` field.
3. `## Decisions Made` — append-only references to ADRs and
   Concept-level material revisions accepted during the session.
4. `## Work Done` — append-only file-touch log with rationale.
5. `## Summary` — 2–4 sentences written once at session end.
6. `## Deferred / Discovered` — optional carry-over items. At
   `Closed`, any unrealised Plan entries are referenced here.

## Mutability

- Body outside `## Goals` and `## Plan` lands without a gate.
- `## Goals` entries are gated at append and on material revision.
  Status transitions (`Active` → `Completed` /
  `Superseded by Goal N`) are atomic and ungated. `Abandoned` is
  gated.
- `## Plan` entries are gated at append and on material revision
  while their successor remains `Planned`. Once the successor
  transitions to `Open`, the entry is frozen and only `Realised
  by` is set atomically.
- `## Summary` is written once at session end; later edits are
  gated.

## Relationships

- Realises the `session` Concept.
- Depends on the `corpus` Trope.
- Composed with `session-start`, `session-end`, `goal-flow`.
