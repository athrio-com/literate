# Category

A category is a closed vocabulary: an explicit list of named members
plus optional morphisms (relations or transitions) between them.
Categories are how LF makes enumerated values visible to authors and
validators. Before a value can be used in prose or code, the value
must be a declared member of its category.

Examples: the Tag set used on ADRs; the ADR Status values
(`Open`, `Accepted`, `Deferred`, `Superseded by ADR-NNN`); the Goal
Status values; the Goal Category values.

## Shape

A category file lives at `corpus/categories/<slug>.md` with a
description, a `## Members` section, optional `## Morphisms` /
`## Transitions`, and `## References`.

## Mutability

- Body is fully mutable.
- Member additions and removals pass the review gate.
- Editorial revisions (clearer wording, examples, references) are
  ungated.
- The `categories.md` index is mechanical and mutable.

## Relationships

- Realised by the `category` Trope, which ships base members
  (`tags`, `adr-status`, `goal-status`, `goal-category`) as typed
  primitives.
