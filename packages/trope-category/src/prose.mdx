# Category Trope

The canonical realisation of the `category` Concept. Declares the
on-disk shape of category files, the member-mutation gating rules,
and the index convention. Ships four base members as typed
primitives.

## File layout

Category files live at `corpus/categories/<slug>.md` with a
description, a `## Members` section, optional `## Morphisms` /
`## Transitions`, and `## References`. The folder index is
`corpus/categories/categories.md`.

## Mutability

- Body is fully mutable.
- Adding or removing a member is a gated authorial change.
- Editorial revisions (clearer wording, examples, references) are
  ungated.
- The `categories.md` index is mechanical and mutable.

## Members shipped

The Trope ships four base members:

- `tags` — base tag vocabulary.
- `adr-status` — ADR Status values and transitions.
- `goal-status` — session-Goal Status values and transitions.
- `goal-category` — session-Goal Category values.

Each member exports both an Effect Schema (for type-checked use in
TS code) and an MDX prose file (for the docs and the vendored
`.literate/`).

Consumers extend by adding their own Member values into their
`corpus/categories/<slug>.md` files.
