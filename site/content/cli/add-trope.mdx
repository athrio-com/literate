# `literate add trope <package>`

Append a Trope package to the consumer's `package.json` `"literate"`
manifest, then run `compile`.

## Synopsis

```
literate add trope <package> [--target <dir>]
```

| Argument | Description |
|---|---|
| `<package>` | Trope package name. v0.1 bundled set: see [`version`](/docs/cli/version). |
| `--target`, `-t` | Target project directory. |

## Behaviour

- Resolves the package against the bundled catalog (v0.1) or the
  consumer's installed npm packages (future).
- If the Trope is already in the manifest, no-op.
- Otherwise: appends, sorts, writes the manifest, calls
  `compileLiterate(target)`.

## Example

```bash
literate add trope @literate/trope-term
literate add trope @literate/trope-chapters
```
