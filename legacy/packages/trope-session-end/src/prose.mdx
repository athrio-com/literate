# Session-End Trope

The workflow Trope that closes every LF session. Validates the log,
stamps the Closed status, manages outstanding Plan entries, and
(when configured) produces a git commit.

## Validations

Before stamping Closed:

1. `## Goals` section exists and contains at least one entry. Every
   entry carries a `Status` (post-Accept). No entry remains
   `Active`.
2. `## Decisions Made` exists.
3. `## Work Done` exists and is populated.
4. `## Summary` exists and is non-empty.
5. `## Deferred / Discovered` exists.
6. If a `## Plan` block exists, every Plan entry is in one of:
   `Realised by …` (a successor session has been started),
   `Planned` (a successor `Planned` log has been created and is
   carried forward in `## Deferred / Discovered`), or
   `Abandoned` (with a one-line rationale beneath the entry).
   No Plan entry may be in an indeterminate state.

On validation fail, the Trope returns
`SessionEndIncomplete` listing what is missing. The session stays
Open until the gaps are resolved.

## Steps

1. Run validations.
2. For each Plan entry without a `Realised by` field whose
   successor `Planned` log exists, append a reference to it under
   `## Deferred / Discovered` so the carry-over is auditable.
3. Stamp the header `Status:` line:
   `Closed (YYYY-MM-DDTHH:MM)` in UTC.
4. Update `corpus/sessions/sessions.md` row.
5. (Optional) Produce a git commit with an imperative subject
   derived from the session slug and Summary.
