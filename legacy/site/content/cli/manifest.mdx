# The `"literate"` manifest

The consumer's manifest is a `"literate"` key inside their project
`package.json`. v0.1 minimum:

```json
{
  "name": "my-project",
  "version": "0.0.0",
  "literate": {
    "version": "0.1.0",
    "tropes": [
      "@literate/trope-corpus",
      "@literate/trope-session",
      "@literate/trope-decisions"
    ]
  }
}
```

## Fields

- `version` — the LF version this project pins to.
- `tropes` — the list of Trope package names to install. Order
  does not matter; the resolver topologically orders them via the
  typed dependency graph.

## Schema

```ts
import { Schema } from 'effect'

export const LiterateManifest = Schema.Struct({
  version: Schema.String,
  tropes: Schema.Array(Schema.String),
})
```

## Beyond v0.1

Future versions will add fields for: per-Trope version pins, local
trope paths (for in-repo experimentation), template registry
sources, and project-local overrides of gating rules.
